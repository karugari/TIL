# Day 08: CSS Layout & Advanced Styling

## 70. 블록 요소 & 인라인 요소의 스타일 차이
요소의 `display` 성질에 따라 레이아웃과 여백 적용 방식이 달라집니다.

* **Block:** 혼자 한 줄을 다 차지함. 가로 너비는 부모의 100%.
* **Inline & Inline-Block:** 옆으로 나란히 배치됨. 가로 너비는 content만큼만 차지함.
* **Inline의 한계:** - 수직 마진을 입력해도 브라우저가 무시함.
    - 수직 패딩 입력 시 배경색은 커지지만 주변 요소를 밀어내지 못함.
    - 부모 요소가 인라인 자식의 높이를 계산하지 못해 요소가 삐져나옴.
* **해결책 (Inline-block):** 나란히 배치되는 성질 + 블록 요소의 상하 여백 기능이 모두 작동함.

> **💡 용도별 선택 가이드**
> - 글자 중간 링크: `inline` 그대로 유지
> - 상하좌우 여백이 있는 예쁜 버튼: `inline-block`
> - 섹션 구분 및 큰 덩어리: `block`

## 71. 마진 상쇄 (Margin Collapsing) 이해하기
* **인라인 요소:** 좌우 여백만 가능. 위아래 여백은 의미 없음.
* **마진 상쇄 법칙:** "두 블록 요소가 수직으로 만날 때, 여백은 합산(+)이 아니라 **최댓값(Max)**을 선택한다!"
* **Display별 차이:**
    - `inline`: 수직 마진 적용 불가.
    - `block`: 마진 겹침(상쇄) 발생.
    - `inline-block`: 마진 겹침 발생 안 함. 설정한 마진이 그대로 적용됨.

## 72. box-shadow 속성 이해하기
요소의 테두리 주변에 그림자를 추가하여 입체감을 줍니다.
* **순서:** X축 오프셋, Y축 오프셋, 블러(Blur), 색상(rgba)
* **특징:** 블러가 `0`이면 선명한 직사각형, 값이 커지면 테두리가 흐려짐. `rgba`의 `alpha`값으로 그림자 농도를 조절.

```css
/* 실습 예시 */
box-shadow: 1px 1px 4px rgba(0, 0, 0, 0.2);
```

## 72-2. 여러분의 챌린지 (Challenge)
직접 코드를 짜보며 레이아웃을 구현하는 실무 연습 세션입니다.
* **텍스트 처리:** 여러 텍스트를 `span`으로 묶고 `display: block`을 주어 영역을 확보함.
* **디자인:** `border-style: solid`, 두께 `4px`, `border-radius: 10px`로 둥근 테두리 스타일링.
* **고급 배치:** 사진과 `span` 요소가 겹치도록 `margin-top: -100px` (음수 마진)을 적용하여 입체적인 레이아웃 완성.

## 72-3. [실습] HTML 레이아웃 만들기
### 💡 강사 vs 나의 방식 차이점 (리마인드)
* **강사:** `main`과 `footer`라는 **시맨틱(Semantic) 레이아웃** 구조를 추가하여 문서의 의미를 명확히 함.
* **나:** 별도의 레이아웃 구조 추가 없이 기존 태그를 활용해 진행함.
* **교훈:** `span`은 의미 없는 인라인 요소이므로 스타일 변경에는 유용하지만, 전체적인 문서 구조를 잡을 때는 `main`, `section` 같은 시맨틱 태그를 쓰는 것이 더 표준적임.

## 74. 메인 섹션 스타일링
* **나의 방식:** 인라인 속성인 `span`을 블록화(`display: block`)시켜 직접 스타일을 입힘.
* **강사의 방식:** `main`이라는 큰 틀을 만들고, 그 구조(Container)에 스타일을 적용하여 하위 요소들을 제어함.
* **학습포인트:** 요소를 개별적으로 스타일링하는 것보다, 큰 레이아웃 단위를 나누어 관리하는 것이 유지보수에 유리함.

## 75. 이미지 스타일링 & 음수 마진 작업
인라인 요소의 종류에 따라 마진이 어떻게 다르게 작동하는지 학습했습니다.

| 구분 | 비대체 인라인 (`<span>`, `<a>`) | 대체 인라인 (`<img>`, `<video>`) |
| :--- | :--- | :--- |
| **특징** | 브라우저 내부의 텍스트 콘텐츠 | 외부 리소스로 내용이 대체되는 요소 |
| **수직 마진** | **작동 안 함** (브라우저가 무시) | **정상 작동** (고유의 크기를 가짐) |



### 음수 마진(Negative Margin)의 원리
* **양수 마진:** 주변 요소를 밀어내며 나를 이동시킴.
* **음수 마진:** 주변 공간을 '먹어 치우며' 나를 해당 방향으로 끌어당김.
* **활용:** 음수 마진을 사용하면 요소가 부모 요소의 경계선(Border) 밖으로 탈출하여 다른 요소와 겹치는 디자인이 가능함.

## 77. 모듈 요약 (HTML & CSS 핵심 지도)

### 1) HTML: 구조와 관계
- **계보 이해:** 부모-자식, 조상-자손 관계에 따라 스타일 상속과 레이아웃이 결정됨.
- **시맨틱 레이아웃:** `main`, `footer`, `header` 등을 사용해 기계(검색엔진)와 사람이 모두 이해하기 쉬운 구조를 만듦.

### 2) CSS 규칙: 스타일 적용 기준
- **상속(Inheritance):** 글꼴, 색상 등은 전달되지만 너비, 마진 등은 전달되지 않음.
- **계단식(Cascading):** 아래쪽에 쓴 코드가 위쪽 코드보다 우선순위가 높음.
- **우선순위(Specificity):** 선택자의 구체성에 따라 점수가 매겨짐.

## 77-2. 모든 CSS 선택자 리스트 (완벽 가이드)

| 선택자 종류 | 구문 | 점수 | 특징 |
| :--- | :--- | :--- | :--- |
| **아이디(ID)** | `#id { }` | **100점** | 페이지당 단 한 번만 사용 (가장 강력) |
| **클래스(Class)** | `.class { }` | **10점** | 여러 요소에 중복 적용 가능 (범용적) |
| **속성(Attr)** | `[src] { }` | **10점** | 특정 속성 보유 여부로 선택 |
| **태그(Tag)** | `p { }` | **1점** | 해당 태그 전체 선택 |
| **범용(Univ)** | `* { }` | **0점** | 모든 요소를 개별적으로 지정 |


### 복합 선택자 활용
- **그룹화:** `p, .class { }` (동일 스타일 일괄 적용)
- **결합:** `p.some-class { }` (p 태그이면서 동시에 해당 클래스인 경우)
- **의사 선택자:** `a:hover { }` (마우스 오버 등 상태에 따른 스타일)

> **Final Remind:** 우선순위 점수가 가장 높은 **ID 선택자(#)**는 태그나 클래스 선택자보다 항상 우선적으로 적용된다!
