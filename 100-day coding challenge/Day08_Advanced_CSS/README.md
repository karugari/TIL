### 블록요소&인라인 요소의 스타일 차이
블록요소는 혼자서 한줄을 다차지하지만 lnline 과 lnline-Block은 옆으로 나란히 배치됨 즉 가로너비는 content만큼 차지함
lnline 요소는 수직마진을 입력해도 브라우저가 아예무시하며
수직 패딩을 입력하면 배경색은 커지지만 주변요소를 밀어낸다
또한 부모 요소가 자식인 인라인 요소의 높이를 제대로 계산하지못해 삐져나온다
이를 해결하기위해 lnline-block을 사용함
lnline-block은 나란이 배치되는 성질과 블록 요소의 상하 여백이 완벽히 작동함

글자 중간에 링크를 넣고 싶을때 : 기본값인 inline 그대로둠
예쁜 버튼(상하좌우 여백이있는)은 lnline-block
섹션을 나누거나 큰덩어리를 만들때: 기본값인 block사용

### CSS 마진 겹침 (Margin Collapsing)
1. 마진 겹침이란?
인접한 두 개 이상의 블록 요소의 **수직 마진(위, 아래)**이 만날 때, 마진 값이 합쳐지지 않고 더 큰 값 하나만 적용되는 현상입니다.

2. 마진 겹침의 3가지 핵심 규칙
인접 요소 사이에서만 발생: 위아래로 나란히 붙어 있는 요소들 사이에서 일어납니다.

수직 방향(Vertical) 한정: 위아래 마진은 겹치지만, 좌우(Horizontal) 마진은 절대 겹치지 않고 합산됩니다.

큰 값이 승리: 두 마진 값이 다를 경우, 더 큰 마진 값이 최종 여백이 됩니다. (예: 20px과 12px이 만나면 여백은 32px이 아니라 20px이 됨)
