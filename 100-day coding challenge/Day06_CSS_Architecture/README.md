## 1. 로컬 개발 환경 (Live Server)
VS Code의 'Live Server' 확장을 통해 내 컴퓨터(Local)를 웹 서버처럼 활용할 수 있다.

### 🌐 IP와 도메인
- **127.0.0.1**: 로컬 컴퓨터(내 컴퓨터)에 예약된 특수 IP 주소.
- **localhost**: `127.0.0.1`의 별칭(Alias). 브라우저 주소창에 입력 시 동일하게 동작한다.

### 🚪 포트 (Port)
- **개념**: 하나의 머신(컴퓨터)에서 여러 개의 프로세스(웹사이트)를 실행하기 위한 출입구.
- **특징**:
  - IP(`127.0.0.1`)는 집 주소라면, 포트(`:5500`)는 방 번호와 같다.
  - 하나의 컴퓨터에서 포트만 다르게 설정하면 3개의 다른 웹사이트를 동시에 호스팅할 수 있다.
  - 예: `localhost:5500`

## 2. HTML 리스트 (Lists)
목록의 성격에 따라 태그를 구분해서 사용하는 것이 의미론적(Semantic)으로 중요하다.

| 태그 | 설명 | 용도 예시 |
|:---:|:---|:---|
| `<ol>` | **Ordered List** (순서가 있는 목록) | 요일, 순위, 레시피 순서 |
| `<ul>` | **Unordered List** (순서가 없는 목록) | 쇼핑 목록, 메뉴 |
| `<li>` | **List Item** (실제 항목) | 목록의 각 요소 |

### 💻 예시 코드
```html
<ol>
    <li>Monday</li>
    <li>Tuesday</li>
</ol>

<ul>
    <li>우유 사기</li>
    <li>운동 하기</li>
</ul>
```
## 3. CSS 박스 모델과 상속 로직

### 부모, 자식, 컨테이너
컨테이너: 다른 요소를 포함하거나 중첩하는 상위 요소.
상속(Inheritance): 부모 요소에 적용된 스타일 규칙이 자식 요소에게도 전달되는 것.

예외: <a> 태그(앵커)는 부모의 색상 등을 강제로 상속받지 않는 경우가 많다.

### 계단식(Cascading) 상속 로직
CSS(Cascading Style Sheets)의 핵심은 여러 규칙이 어떻게 적용되는지 결정하는 것이다.

상속 (Inheritance): 특정 속성은 부모에서 자식으로 내려온다.
순서 (Order): 동일한 가중치의 규칙이 여러 개일 경우, 가장 마지막(최신)에 작성된 규칙이 적용된다.

### 명시성/우선순위 (Specificity):

더 구체적인 선택자(Selector)가 덜 구체적인 선택자보다 우선한다.
예: div 태그 선택자보다 id="box" 선택자가 더 강력하다.

📝 요약
"계단식 상속은 하나의 요소에 여러 규칙이 올 수 있음을 의미하며, 충돌 시 더 구체적인 것이, 그마저 같다면 더 나중에 쓴 것이 이긴다."
